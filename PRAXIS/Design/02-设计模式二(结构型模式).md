# 代理模式

概念：为其他对象提供一种代理以控制对此对象的访问

代理：一个具有与被代理对象相同接口的类，客户端只有通过 Proxy 来实现与被代理类的交互，并且在交互过程中 ，代理可以增加其它操作，如：
- 远程代理：隐藏一个对象在不同地址空间的事实
- 虚拟代理：存放需要很长时间实例化的对象
- 安全代理：控制真实对象的访问权限
- 智能引用；当调用真实对象时，代理处理另外一些事


优点：
- 职责清晰
- 高拓展
- 智能化
  
缺点：
- 增加系统复杂度
- 请求速度变慢

```cpp
// 主题接口
class Subject {
public:
    virtual ~Subject() {}
    virtual void request() = 0;
};

// 真实主题
class RealSubject : public Subject {
public:
    void request() override {
        std::cout << "RealSubject: handling request" << std::endl;
    }
};

// 代理类
class Proxy : public Subject {
private:
    std::unique_ptr<RealSubject> realSubject_;
    
    void checkAccess() {
        std::cout << "Proxy: checking access" << std::endl;
    }
    
    void logAccess() {
        std::cout << "Proxy: logging access" << std::endl;
    }
    
public:
    void request() override {
        checkAccess();        // 拓展新功能
        if (!realSubject_) {  // 代理真实主题
            realSubject_ = std::make_unique<RealSubject>();
        }
        realSubject_->request();
        logAccess();
    }
};

```

# 适配器模式

概念：属于包装模式。可以改变类的接口形式，将一个类的接口转换成另一种形式的接口，兼容不同接口的类

优点：
- 提供类的复用性
- 增加类的透明性
- 高灵活性

缺点：
- 增加系统复杂性
- 使用过多易导致系统可读性差

```cpp
// 目标接口
class Target {
 public:
  virtual ~Target() {}
  virtual void request() = 0;
};

// 适配者类
class Adaptee {
 public:
  void specificRequest() {
    std::cout << "Adaptee: specific request" << std::endl;
  }
};

// 适配器
class ObjectAdapter : public Target {
 private:
  std::unique_ptr<Adaptee> adaptee_;

 public:
  ObjectAdapter() : adaptee_(std::make_unique<Adaptee>()) {}

  explicit ObjectAdapter(std::unique_ptr<Adaptee> adaptee)
      : adaptee_(std::move(adaptee)) {}

  void request() override {
    std::cout << "Adapter: translating request..." << std::endl;
    adaptee_->specificRequest();
  }
};
```

# 桥接模式

概念：使用封装、聚合及继承等行为让不同的类承担不同的责任，将抽象与行为实现分离

优点：
- 分离了接口和实现
- 提高可扩展性
- 符合开闭原则

缺点：
- 增加了系统的理解和设计难度
- 需要正确识别出系统中两个独立变化的维度

```cpp
class Implementor {
 public:
  virtual ~Implementor() = default;
  virtual void operationImpl() = 0;
};

class ConcreteImplementorA : public Implementor {
 public:
  void operationImpl() override {
    std::cout << "ConcreteImplementorA operation" << std::endl;
  }
};

class ConcreteImplementorB : public Implementor {
 public:
  void operationImpl() override {
    std::cout << "ConcreteImplementorB operation" << std::endl;
  }
};

class Abstraction {
 protected:
  std::unique_ptr<Implementor> implementor_;

 public:
  explicit Abstraction(std::unique_ptr<Implementor> impl)
      : implementor_(std::move(impl)) {}

  virtual ~Abstraction() = default;

  virtual void operation() { implementor_->operationImpl(); }
};

class RefinedAbstraction : public Abstraction {
 public:
  using Abstraction::Abstraction;

  void operation() override {
    std::cout << "RefinedAbstraction: ";
    implementor_->operationImpl();
  }
};
```
