# 指针与引用

|        | 引用         | 指针     |
| ------ | ------------ | -------- |
| 语法   | `int&`       | `int*`   |
| 初始化 | **必须**     | 可以不   |
| 赋值   | 不能为空     | 可以为空 |
| 指向   | 不能更换目标 | 可以更换 |

## 引用

作用：给变量起别名 等价于 指针常量（`int * const p;`）  

- 左值引用：常规引用，表示对象的身份
  避免参数拷贝
- 右值引用： 必须绑定到右值(临时对象)的引用，表示值
    可实现转移语义和精确传递，达到消除不必要的对象拷贝和更简洁明确地定义泛型的目的

## this 指针

作用：
  - 不影响对象大小
  - 作用域在类内部，自动添加，传递对象本身的地址

使用：
  - 返回类对象本身:`return *this` 
  - 区分成员变量:`this->`

# const

## 1.修饰变量

- 定义常量，同宏定义类似
  **宏常量**：`#define 常量名 常量值`  
**常类型**：使用类型修饰符**const**说明的类型，值不能被改变, `const  数据类型 常量名 = 初始值`;  
- 类型检查
   1. `const`常量具有类型，编译器可以进行安全检查
   2. 宏常量具体类型由字面量和运算符结果类型相关，不进行安全检查
   3. `const`定义的变量只有类型为整数或枚举，且以常量表达式初始化才能作为常量表达式
- 防止修改

注：常量初始化后不能修改，且定义时必须初始化  

`const` 在确定初始值时是编译器常量，当然在某些情况也有可能退化为运行时常量   
`constexpr` 更加严格，编译器保证 `constexpr` 必须是编译期常量，并且 `constexp`r 可修饰函数使其在编译期计算。所以 `constexpr` 更像是 `#define`

注：`mutable` 关键字可以使常函数修改参数  

注：`const`对象默认为文件局部变量,非`const`变量默认为`extern`。要使`const`变量能够在其他文件中访问，必须在文件中显式地指定它为`extern`

## 2.修饰指针

```cpp
const int* a;         // 指向常量的指针，底层const - 值不能改变 
int * const a;        // const 指针,顶层const     - 指向不能改变 
const int * const a;  // 指向const对象的const指针

```

注：
  - 所谓值不能改变是指不能通过指针进行修改
  - `const`指针必须进行初始化

## 3.修饰函数

### 修饰函数返回值

```cpp
const int func();  // 无意义
const int* func(); // 值不变
int* const func(); // 指向不变
```

### 修饰函数参数

```cpp
void func(const int var);  // 传入形参，无意义
void func(int* const var); // 同上

void func(const int* var); // 防止修改变量
void func(const string& s); // 增加效率同时防止修改
```

注；仅对非内部数据类型的输入参数，将“值传递”的方式改为“const 引用传递”，可提高效率

## 4.修饰类

任何不会修改数据成员的函数都应该声明为**`const`类型**,称**常成员函数**

注：
- 只有常成员函数才有资格操作常量或常对象，没有使用**const**关键字进行说明的成员函数不能用来操作常对象
- 类中的`const`成员变量必须通过初始化列表进行初始化
- `const`对象只能访问`const`成员函数,而非`const`对象可以访问任意的成员函数,包括`const`成员函数

# static

由 `static` 关键字修饰的变量称静态变量，其具备共享数据、编译时分配内存、类内声明，类外初始化、类名可直接访问的特点，其中静态成员函数只能访问静态成员变量  

当变量使用 `static` 关键字修饰时，空间将在**程序的生命周期**内分配，且只**分配一次**

注：类中的静态变量由**对象**共享，但不能使用***构造函数***初始化

注；`static` 关键字修饰的变量将限定访问范围在本文件当中

# 多态

## 静态多态，动态多态（virtual）  

函数重载即为静态多态  
静态多态编译时绑定，动态多态运行时绑定  

动态多态 - 指针vfptr
1. 继承时子类重写父类虚函数
2. 父类指针或引用指向子类对象（base &p = son）  

虚析构解决子类堆区释放问题，纯虚析构也需要实现但抽象类无法实例  

虚继承解决菱形基础 (virtual)

## 虚函数

使用 `virtual` 修饰的函数

虚函数的调用取决于指向或者引用的对象的类型，而不是指针或者引用自身的类型

注；
  - 虚函数不能被 `static` `const` 和 `volatile` 关键字
  - 构造函数不能声明为虚函数
    vptr 在运行阶段实例化对象时才产生
  - 析构函数可以声明为虚函数
  - 当虚函数表现多态性的时候不能内联

## 纯虚函数

纯虚函数：即没有函数体的虚函数
抽象类：包含纯虚函数的类
语法：`virtual void func() = 0;`

注：抽象类只能作为基类来派生新类使用，不能创建抽象类的对象

## 虚表机制

C++ 的动态多态性通过虚函数实现，即指向子类的基类指针可以调用子类的函数
```cpp
Base *pt = new Derived();
pt->DerFunc(); 
```

C++ 使用虚拟表实现虚函数，即**每个含虚函数的类都具备一个虚拟表**，虚拟表中包含虚函数指针，指向可访问的类派生函数。同时还拥有一个指向基类的指针**vptr**，创建实例时自动指向类的虚拟表

注：vptr 由派生类继承

# 宏

## 特殊符号
```cpp
/* 字符串化 - #  */
// 
#define exp(s)    // exp("Helo")
#define exp1(#s)  // exp1(Helo)

// 当传入参数名间存在空格时，自动以一个空格连接，忽略剩余空格


/* 符号连接 - ## */
// 先分隔，然后进行强制连接

#define exp(s) hello_##s
exp(world) // hello_world 

/*
1.当用##连接形参时，##前后的空格可有可无。
2.连接后的实际参数名，必须为实际存在的参数名或是编译器已知的宏定义。
3.如果##后的参数本身也是一个宏的话，##会阻止这个宏的展开。
*/

/* 续行 - \ */
// 表示下一行继续定义同一宏

#define MAX(a,b) ((a) >(b) ? (a) \
        :(b))
```

# 内存分区规则  

代码区：存放函数体的二进制代码   
1. 存放CPU执行的机器指令
2. 区域共享
3. 只读  
    
全局区：全局变量和静态变量及常量  
栈区：自动分配释放，函数的参数值和局部变量  
堆区：程序员手动分配释放的空间   

## 位域

作为一种数据结构，可以把数据以位的形式紧凑的储存，并允许程序员对此结构的位进行操作

- 位域在内存中的布局是与机器有关的
- 位域的类型必须是整型或枚举类型，带符号类型中的位域的行为将因具体实现而定
- 取地址运算符（&）不能作用于位域，任何指针都无法指向类的位域

```cpp
struct bit_field_name
{
    type member_name : width;
};
```
| Elements       | Description                       |
| -------------- | --------------------------------- |
| bit_field_name | 位域结构名                        |
| type           | int、signed int 或者 unsigned int |
| member_name    | 位域成员名                        |
| width          | 规定成员所占的位数                |

# 类

- 空类大小为 1 字节
- 函数及静态变量不占用类存储空间
- 虚函数只占一个 vptr 指针空间，继承基类 vptr
